Getting Started

From zero to your first skill in 5 minutes

Everything you need to set up the repo, run a built-in example, and create your first custom AI agent skill.

Phase 1 needs no API key. Claude Code uses your Claude subscription. For Phase 2, copy .env.example to .env, add your key, then set up a Python venv:

cp .env.example .env
# Edit .env: replace your-api-key-here with your key from platform.claude.com/settings/api-keys

# Create a venv, activate it, and install the SDK
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install claude-agent-sdk

# Load the API key into your environment
source .env

Setup

  1. Clone the repo

    git clone https://github.com/sinned/prototype-with-claude
cd prototype-with-claude

  2. Open in Claude Code

    claude

    Claude Code will read CLAUDE.md automatically. This file tells Claude how to help you in this repo — how to design skills, generate SDK code, and what patterns to follow.

  3. Try a built-in skill

    The repo ships with four skills already installed in .claude/skills/. Try one now:

    /research-agent Stripe

    Claude will search the web, read pages, and save a brief to output/stripe-brief.md.

    Other built-in skills to try:

    /qualify-lead Notion
/generate-content "cold email for a sales tool targeting VP Sales"
/qualify-lead-demo Retool   # deliberately weak skill — used to demo the eval loop

  4. Check the output

    # From the terminal
cat output/stripe-brief.md

    Or just ask Claude to show you what was produced.

Build your own skill

The fastest path to your first custom skill:

  1. Run the wizard

    /new-skill

    Claude will ask you: What is the job? What are the inputs? What should it produce? It creates the SKILL.md for you.

  2. Test it with 3 real inputs

    /your-skill-name example-input-1
/your-skill-name example-input-2
/your-skill-name edge-case-input

    Real inputs reveal problems that obvious test cases don't. Use inputs from your actual use case.

  3. Iterate on the SKILL.md

    When the output isn't right, open .claude/skills/your-skill/SKILL.md and make the instructions more specific. Then re-run. This is the core prototyping loop.

    The most common fix: vague instructions → specific ones. Replace "research the company" with "search for the company name, their pricing page URL, and their most recent funding news."

  4. Add test cases and run evals

    # Create test cases
mkdir -p .claude/skills/your-skill/evals
# Copy the example and fill in your test inputs + criteria
cp examples/04-evals-and-improvement/test-cases-example.md \
   .claude/skills/your-skill/evals/test-cases.md

# Run evals
/eval-skill your-skill

  5. Improve until it passes

    /improve-skill your-skill

    Claude reads the eval report and edits your SKILL.md to fix the failures. Re-run /eval-skill to measure the improvement.

  6. Export to production code

    /export-to-sdk

    Claude generates sdk/your-skill/agent.py — working Python code that runs the same job autonomously using the Agent SDK.

The mental model

There are three things to understand:

SKILL.md
A job description for your AI. It contains a description (when to trigger), steps (what to do), and output format (what to produce). Claude Code reads it when you invoke the command. Plain markdown — no code.
query()
That same job, running autonomously. The Agent SDK's query() function takes a prompt (your skill instructions) and runs the agent loop — same tools, same Claude — without a human in the loop.
evals
How you know it works. Test cases with binary criteria (yes/no) graded by Claude. Not "does it look good" — "does it include a source URL?" You grade outputs against criteria, find patterns in failures, improve the skill.
