Examples

Three patterns. Four examples.

Each example includes a complete Skill and a production Agent SDK migration. Copy, modify, and ship.

The three skill patterns

Most startup AI features fit one of these shapes. Pick the closest one and adapt it.

01
Research & Synthesize
Shape: Input → Search → Gather → Synthesize → Structured output

Tools: WebSearch, WebFetch, Write

Use when the job is gathering information from multiple sources and packaging it clearly.
02
Score & Classify
Shape: Input → Load rubric → Analyze → Score → Decision + reasoning

Tools: Read, Write, WebSearch

Use when the job is applying a consistent rubric to an input to produce a categorized output.
03
Generate & Iterate
Shape: Input → Research → Plan variations → Generate all → Output set

Tools: Read, Write, WebSearch

Use when the job is producing content where having multiple options has value.
Research & Synthesize

01 — Research Agent

Research any company, person, or topic and produce a structured intelligence brief saved to a file.

The Skill (prototype)

File: .claude/skills/research-agent/SKILL.md

---
name: research-agent
description: Research a company, person,
  or topic and produce a brief. Use when
  asked to research or learn about
  something.
---

# Research Agent

## Steps
1. Run 3-5 targeted searches
2. Fetch 2-3 most relevant URLs
3. Synthesize: 5-7 key insights
4. Write brief to output/-brief.md
5. Print one-paragraph summary

## Output format
# [Subject] — Intelligence Brief
## TL;DR / Overview / Key Facts
## Recent Developments / Sources

Run with: /research-agent Stripe

The SDK (production)

File: examples/01-research-agent/sdk/agent.py

from claude_agent_sdk import (
  query, ClaudeAgentOptions
)

async def research(subject: str):
    async for message in query(
        prompt=PROMPT.format(
            subject=subject
        ),
        options=ClaudeAgentOptions(
            allowed_tools=[
                "WebSearch",
                "WebFetch",
                "Write",
                "Bash",
            ],
            permission_mode="acceptEdits",
        ),
    ):
        if hasattr(message, "result"):
            return message.result

Run with: python agent.py "Stripe" or python agent.py --batch companies.txt

Score & Classify

02 — Lead Qualifier

Research a company and score it against your Ideal Customer Profile. Returns Hot / Warm / Cold / Disqualified with a scorecard and recommended next step.

Key concept: configurable rubric

Create icp.md in your project root:

# My ICP

## Hard criteria (disqualify if not met)
- Must be B2B
- At least 20 employees

## Soft criteria
- Size: 50-500 employees
- Series A through Series C

## Strong positive signals
- Recently raised funding
- Hiring sales roles
- Fast headcount growth

The skill reads icp.md automatically. Change the file → change the scoring criteria.

Output format

Saved to output/leads/<company>-qualification.md

# Lead Qualification: Notion
Tier: WARM

## Decision
WARM — Strong B2B SaaS fit, but
headcount above ideal range.

## ICP Scorecard
| Criterion    | Score | Evidence         |
|--------------|-------|------------------|
| Size         |  ⚠️   | ~800 employees   |
| Industry     |  ✅   | B2B SaaS         |
| Geography    |  ✅   | US-headquartered |
| Stage        |  ✅   | Post Series C    |
| Pain signals |  ✅   | 12 open AE roles |

## Next step
Book discovery — mention their
recent Series C and AE expansion.

Adapting to other scoring use cases

The rubric pattern works for anything you'd score against criteria:

Job application screening Support ticket routing Investment deal screening Content moderation Partner vetting

Generate & Iterate

03 — Content Generator

Input a content brief. Claude produces 3 variations with different angles (story-led, data-driven, contrarian) plus a recommendation for which to use.

Brand voice configuration

Create brand-voice.md:

# Brand Voice

## Tone
Direct, confident, human.
No buzzwords or corporate filler.

## Avoid
- "leverage" (as a verb)
- "utilize" (use "use")
- "cutting-edge"
- Passive voice

## Good example
"We built a faster way to run team
retrospectives. Here's what's new."

## Bad example
"We are excited to announce our
revolutionary platform that leverages
cutting-edge AI technology..."

What you get

3 files in output/content/<slug>/

variation-a.md — Story-led
Opens with a customer pain point or founder story
variation-b.md — Data-driven
Leads with a statistic or benchmark
variation-c.md — Contrarian
Challenges a common assumption
brief-summary.md
Comparison table + which to use and why

How to adapt an example to your use case

  1. Copy the example skill

    mkdir -p .claude/skills/my-skill
cp examples/01-research-agent/SKILL.md .claude/skills/my-skill/SKILL.md

  2. Change the name and description

    Edit the frontmatter. The name becomes your slash command. The description controls when Claude auto-triggers it.

  3. Modify the steps for your job

    The steps are the most important part. Make them specific to your use case. What exactly should Claude search for? What does "done" look like?

  4. Customize the output format

    Change the file path, the sections, the structure. The more specific you are here, the more consistent the outputs.

  5. Test, eval, improve

    Run it on 5 real inputs. Add test cases. Run /eval-skill. Run /improve-skill. Repeat.
