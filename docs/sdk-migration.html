Ship It — Agent SDK Migration

Your validated skill, now in production

The Agent SDK gives you the same Claude, the same tools, and the same instructions — now running autonomously in your app.

When to migrate

Migrate from a Skill to the Agent SDK when any of these are true:

Automated pipelines
You need the agent to run without a human present — cron jobs, webhooks, background tasks.
Batch processing
You need to process many inputs — a CSV of leads, a list of articles, a queue of tickets.
Product integration
You need to embed the agent in your web app, mobile app, or API.
Multi-agent systems
You need to chain agents together or run specialized subagents in parallel.

Don't migrate prematurely. If you're still figuring out what the AI should do, keep prototyping in Claude Code. The migration is easy — premature migration is just friction.

The migration is simpler than you think

The core of the migration: copy your SKILL.md instructions into a Python prompt template.

Before — SKILL.md

---
name: research-agent
description: Research a company
  and produce a brief.
---

# Research Agent

## Steps
1. Search for {topic} using
   3 targeted queries
2. Read the top 3 results
3. Write brief to
   output/{topic}-brief.md

After — agent.py

PROMPT = """
Research {topic}.

Steps:
1. Search for {topic} using
   3 targeted queries
2. Read the top 3 results
3. Write brief to
   output/{slug}-brief.md
"""

async def run(topic: str):
    async for msg in query(
        prompt=PROMPT.format(
            topic=topic,
            slug=slugify(topic)
        ),
        options=ClaudeAgentOptions(
            allowed_tools=[
                "WebSearch",
                "WebFetch",
                "Write",
            ],
        ),
    ):
        if hasattr(msg, "result"):
            return msg.result

The instructions are identical. Only the container changed.

SDK patterns

1. Single input → single output (the baseline)

import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions

async def run_agent(input_data: str) -> str | None:
    result = None
    async for message in query(
        prompt=f"Do the job with: {input_data}",
        options=ClaudeAgentOptions(
            allowed_tools=["WebSearch", "Write"],
            permission_mode="acceptEdits",
        ),
    ):
        if hasattr(message, "result"):
            result = message.result
    return result

if __name__ == "__main__":
    print(asyncio.run(run_agent("your input")))

2. Batch processing (sequential)

async def run_batch(items: list[str]) -> list[dict]:
    results = []
    for i, item in enumerate(items, 1):
        print(f"[{i}/{len(items)}] Processing: {item}")
        result = await run_agent(item)
        results.append({"input": item, "result": result})
    return results

3. Batch processing (parallel, with concurrency limit)

import asyncio

async def run_batch_parallel(
    items: list[str],
    max_concurrent: int = 3,
) -> list[dict]:
    semaphore = asyncio.Semaphore(max_concurrent)

    async def run_one(item: str) -> dict:
        async with semaphore:
            result = await run_agent(item)
            return {"input": item, "result": result}

    return await asyncio.gather(*[run_one(item) for item in items])

4. Load a config file before running

from pathlib import Path

def build_prompt(task: str) -> str:
    # Load ICP, brand voice, or any other config
    config = Path("icp.md").read_text() if Path("icp.md").exists() else ""
    return f"""
Context / criteria:
{config}

Task: {task}
"""

5. Multi-step with session continuity

from claude_agent_sdk import SystemMessage

async def run_pipeline(steps: list[str]) -> list[str]:
    session_id = None
    results = []

    for i, prompt_text in enumerate(steps):
        options = ClaudeAgentOptions(
            allowed_tools=["Read", "Write"],
            resume=session_id if session_id and i > 0 else None,
        )
        async for message in query(prompt=prompt_text, options=options):
            if isinstance(message, SystemMessage) and message.subtype == "init":
                session_id = message.data.get("session_id")
            if hasattr(message, "result"):
                results.append(message.result)

    return results

6. Subagents for specialized parallel work

from claude_agent_sdk import AgentDefinition

async def run_with_subagents(topic: str) -> str | None:
    result = None
    async for message in query(
        prompt=f"Research {topic} — use web-researcher for data, analyst for synthesis",
        options=ClaudeAgentOptions(
            allowed_tools=["Agent", "Write"],
            agents={
                "web-researcher": AgentDefinition(
                    description="Searches and fetches web pages",
                    prompt="Search for information and return what you find",
                    tools=["WebSearch", "WebFetch"],
                ),
                "analyst": AgentDefinition(
                    description="Synthesizes information into structured briefs",
                    prompt="Analyze the provided information and produce a structured brief",
                    tools=["Read", "Write"],
                ),
            },
        ),
    ):
        if hasattr(message, "result"):
            result = message.result
    return result

Embedding in a web app

The most common production deployment: a FastAPI endpoint that triggers the agent.

Synchronous endpoint

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class AgentRequest(BaseModel):
    input: str

@app.post("/run")
async def run_endpoint(request: AgentRequest):
    result = await run_agent(request.input)
    return {"result": result, "status": "done" if result else "failed"}

Async endpoint (for long-running jobs)

import uuid
from fastapi import BackgroundTasks

jobs: dict[str, dict] = {}

@app.post("/run-async")
async def run_async(request: AgentRequest, background_tasks: BackgroundTasks):
    job_id = str(uuid.uuid4())
    jobs[job_id] = {"status": "running", "result": None}

    async def do_work():
        result = await run_agent(request.input)
        jobs[job_id] = {"status": "done", "result": result}

    background_tasks.add_task(do_work)
    return {"job_id": job_id}

@app.get("/jobs/{job_id}")
async def get_job(job_id: str):
    return jobs.get(job_id, {"status": "not_found"})

Permission modes

ModeWhat it meansUse when
"default" Prompts for approval on tool use Development, interactive use
"acceptEdits" Auto-approves file edits Production agents that write files but don't run shell commands
"bypassPermissions" Auto-approves everything Fully trusted, audited agents only

Recommendation: Use "acceptEdits" for most production agents. Only use "bypassPermissions" after a thorough security review of your prompts and tool access.

Keep your Skill and SDK code in sync

The prompt in agent.py and the instructions in SKILL.md should always match. Convention to enforce this:

# Prompt migrated from: .claude/skills/research-agent/SKILL.md
# When you update the skill, update this template too.
PROMPT_TEMPLATE = """
Research {subject} and produce a structured intelligence brief.

Steps:
1. Run 3-5 targeted searches...
"""

When you tune the SKILL.md (via /improve-skill or manually), update the matching PROMPT_TEMPLATE constant. The Skill is the source of truth — the SDK code is its production runtime.
